data-transformation-pipeline is a workflow with execution model batch and topology pattern linear. Its purpose is: Data transformation pipeline using Google Cloud Dataform for data processing with parameter parsing, compilation, workflow invocation, and state monitoring.

Scheduling: schedule type is event_driven, catchup/backfill is False. Control flow includes dependencies such as: start→parse_input_params; parse_input_params→create_compilation_result; create_compilation_result→create_workflow_invocation; create_workflow_invocation→is_workflow_invocation_done; is_workflow_invocation_done→end.

start performs Pipeline start marker using mechanism unknown. parse_input_params performs Parse input parameters and prepare compilation configuration using mechanism python_callable. create_compilation_result performs Create Dataform compilation result with dynamic parameters using mechanism python_callable. create_workflow_invocation performs Create and trigger Dataform workflow invocation using mechanism unknown. is_workflow_invocation_done performs Monitor Dataform workflow invocation completion status using mechanism wait_poll. end performs Pipeline completion marker using mechanism unknown.
