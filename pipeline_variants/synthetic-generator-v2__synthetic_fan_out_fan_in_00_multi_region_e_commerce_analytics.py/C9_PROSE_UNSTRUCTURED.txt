multi_region_ecommerce_analytics is a workflow with execution model batch and topology pattern fan_out_fan_in. Its purpose is: Ingest regional sales data, convert currencies to USD, and generate global revenue report.

Scheduling: schedule type is cron, expression is @daily, start date is 2024-01-01, catchup/backfill is False. Control flow includes dependencies such as: start_pipeline→ingest_us_east; start_pipeline→ingest_us_west; start_pipeline→ingest_eu; start_pipeline→ingest_apac; ingest_us_east→convert_currency_us_east; ingest_us_west→convert_currency_us_west; ingest_eu→convert_currency_eu; ingest_apac→convert_currency_apac; convert_currency_us_east→aggregate_global_revenue; convert_currency_us_west→aggregate_global_revenue; convert_currency_eu→aggregate_global_revenue; convert_currency_apac→aggregate_global_revenue.

start_pipeline performs Start the pipeline execution using mechanism unknown. ingest_us_east performs Ingest sales data from US-East region using mechanism python_callable. ingest_us_west performs Ingest sales data from US-West region using mechanism python_callable. ingest_eu performs Ingest sales data from EU region using mechanism python_callable. ingest_apac performs Ingest sales data from APAC region using mechanism python_callable. convert_currency_us_east performs Convert US-East sales data currency to USD using mechanism python_callable. convert_currency_us_west performs Convert US-West sales data currency to USD using mechanism python_callable. convert_currency_eu performs Convert EU sales data currency to USD using mechanism python_callable. convert_currency_apac performs Convert APAC sales data currency to USD using mechanism python_callable. aggregate_global_revenue performs Aggregate all regional converted data into global revenue report using mechanism python_callable. end_pipeline performs End the pipeline execution using mechanism unknown.
